# 子供向け学習Webアプリ — 最終企画書

> **プロジェクト名**: そだてる ずかん（仮称）
> **ターゲット**: 3〜6歳の子供（＋保護者）
> **策定日**: 2026-02-28
> **策定方法**: 6名AIチームによる3ラウンド議論シミュレーション

---

## 1. コンセプト

### 「育てる図鑑 × きもちの冒険」

学ぶたびに図鑑のページが増え、キャラクターに「きもちのことば」を教えてあげる体験。
コレクションの喜びと、感情教育（SEL）を自然に融合させた学習アプリ。

**コアバリュー**:
- 子供が「教える側」になることで学びが定着する（プロテジェ効果）
- 正解を押し付けず、共感と発見で動機づける
- 親子の関わりを技術で補助する

## 2. PM最終決定事項（6つの対立点の決着）

| # | 対立テーマ | 最終決定 | 理由 |
|---|-----------|---------|------|
| 1 | 学習領域 | **ひらがな主軸＋感情ことばを副次統合** | 保護者の購買動機（ひらがな）と教育的価値（SEL）を両立 |
| 2 | 正解の有無 | **ひらがな=正解あり / 感情=正解なし** | ひらがなは効果測定可能。感情は「否定せず視野を広げる」方式 |
| 3 | プロトタイプ期間 | **3週間MVP＋法務レビュー並行** | 動くものがないと議論が空転する。法務は同時進行 |
| 4 | 図鑑の表現 | **静止画＋簡易トランジション（初期）** | 3週間で品質の高いアニメは不可。反応を見て追加投資 |
| 5 | プロテジェ効果 | **全年齢導入、5-6歳で重点検証** | 「教えるUI」は年齢問わず有効だが効果測定は5-6歳から |
| 6 | ビジネスモデル | **基本無料→フリーミアム（月額1,500円）** | 初期はデータ収集優先。課金は利用データで根拠を作ってから |

**記録された反対意見**:
- マナブ: ひらがな正解ありが外発的動機に偏るリスクを懸念（次回スプリントで再評価）
- ケンジ: 3週間では法的準備が不十分（並行対応で合意）

## 3. MVP仕様

### 3.1 学習モジュール（1本）

**ひらがな × きもちことば 複合モジュール**

```
フロー例:
1. キャラ（くまの「モコ」）が画面に登場
2. モコが「今日は"あ"のつくことばを探そう！」と音声で話す
3. 選択肢が3つ表示（絵＋ひらがな）→ タップで回答
4. 正解 → 図鑑ページが1枚開く！（静止画＋トランジション）
5. 開いた図鑑のページでモコが気持ちを表現
   「あめが降って、モコはちょっとかなしいな...」
6. 子供に問いかけ:「モコにどんなきもちを教えてあげる？」
7. 感情アイコン（3択）をタップ → どれを選んでもモコが反応
   「そうなんだね、教えてくれてありがとう！」
8. 1セッション = 5問（約5分）→ タイマーで自動終了
```

### 3.2 年齢別設計

| 要素 | 3-4歳 | 5-6歳 |
|------|-------|-------|
| ひらがな | 5文字（あいうえお） | 10文字 |
| 感情選択肢 | 2択（うれしい/かなしい） | 4択（+こわい/おこった） |
| キャラの問いかけ | 「えらんでね」 | 「どうしてそう思う？」 |
| 図鑑ページ | 5ページ | 10ページ |
| UIサイズ | 特大ボタン | 大ボタン |

### 3.3 保護者向け機能

- 進捗ダッシュボード（今日覚えたひらがな、開いた図鑑ページ数）
- 週次チェックリスト通知:「アプリで出た"きもちのことば"、家で使いましたか？」
- スクリーンタイム設定（1日の上限回数）
- 匿名ID管理（個人情報不要）

### 3.4 実装しないもの（MVP除外）

- ❌ 音声認識
- ❌ AI個別最適化
- ❌ 英語 / STEM
- ❌ アニメーション（リッチなもの）
- ❌ 課金機能
- ❌ ソーシャル機能 / ランキング
- ❌ 手書き入力

## 4. 技術スタック（テックリード テツヤ承認済み）

| レイヤー | 技術 | 選定理由 |
|---------|------|---------|
| フロントエンド | **Next.js (App Router)** | PWA対応、SSG可能、開発速度 |
| アニメーション | **Framer Motion** | タップ操作・トランジションに十分 |
| バックエンド | **Firebase** (Auth + Firestore) | 匿名ログイン対応、BaaS完結で運用負荷最小 |
| ホスティング | **Firebase Hosting** or **Vercel** | CDN配信で表示速度担保 |
| 音声再生 | **Web Audio API** | ブラウザ標準、追加依存なし |
| 素材管理 | **Firebase Storage** | イラスト・音声ファイルの配信 |

### テツヤの技術評価

> **「この案は"技術で冒険しない正しい企画"だ。Go判断を出していい。」**
>
> - エンジニア2名で2週間でコアモジュール完成可能
> - 最大のボトルネックは素材（イラスト・音声）の納品
> - プレースホルダで先行開発し、差し替え可能な設計にする
> - 初期は5-6歳向け1本に集中し、3-4歳版は後追い推奨

## 5. 教育的裏付け（教育専門家 マナブ承認済み）

### 教育的妥当性: **8.5 / 10**

| 理論 | 適用箇所 |
|------|---------|
| **ピアジェ 前操作期** | シンボルと言語の獲得段階に合わせたひらがな＋感情語彙 |
| **ヴィゴツキー ZPD** | 少し背伸びした課題を自分のペースで攻略 |
| **プロテジェ効果** | キャラに「教えてあげる」ことで学びが定着 |
| **SEL（社会情動的学習）** | 感情語彙の習得が認知発達の土台になる |
| **内発的動機づけ** | 感情パートは正解なし、共感と受容がベース |

### 感情パートの設計原則

> 「否定せず、視野を広げる」方式
> - 子供がどの感情を選んでも「そうなんだね、教えてくれてありがとう」と受容
> - その上でキャラが「ぼくはこう感じるよ」と別の視点を提示
> - テストではなく対話。数値化ではなく日常般化で効果測定

### 効果測定方法

- 保護者向け週次チェックリスト:「アプリで出た感情の言葉を日常で使いましたか？」
- 図鑑コンプリート率 × セッション継続率の相関分析
- ひらがな正答率の推移（ひらがなパートのみ）

## 6. リスク管理（QA ケンジ承認条件）

### 絶対条件（ローンチ前必須）

| # | 条件 | 担当 | 期限 |
|---|------|------|------|
| 1 | 親権者同意メカニズムの実装 | テツヤ + 法務 | ローンチ前 |
| 2 | 利用規約・プライバシーポリシーの弁護士レビュー | ケンジ + 法務 | MVP完成後2週間以内 |
| 3 | 5分タイマーの確実な動作（子供が無視できない設計） | ハルカ + テツヤ | MVP内 |
| 4 | 個人情報保護方針の適合性確認 | ケンジ | ローンチ前 |
| 5 | テスト参加親子の多様性確保（年齢・背景のバランス） | ユウキ | テスト前 |

### プロトタイプテスト必須項目

- [ ] UI操作の直感性（タップ誤操作率）
- [ ] 感情ことばの理解度・受け入れ度
- [ ] 親からの満足度・懸念点の詳細ヒアリング
- [ ] デバイス・ネット環境への耐性
- [ ] セッション完了率（5問中何問まで到達するか）
- [ ] 図鑑を開いた時の子供の反応（喜び・無関心・困惑）

### ローンチ前法的チェックリスト

- [ ] 利用規約の弁護士レビュー完了
- [ ] プライバシーポリシーの策定・公開
- [ ] 個人情報保護法への適合確認
- [ ] 特定商取引法に基づく表示（課金導入時）
- [ ] 未成年利用に関する親権者保護条項
- [ ] データ保持期間・削除ポリシーの明確化

## 8. スケジュール

```
Week 1-2: 開発フェーズ1
  ├── テツヤ: Next.js基盤、Firebase設定、コアUI
  ├── ハルカ: UIデザイン、フロー設計、プレースホルダ素材
  ├── マナブ: 学習コンテンツ作成（ひらがな5文字＋感情語彙）
  └── ケンジ: 法務チェックリスト作成、弁護士選定

Week 3: 開発フェーズ2 + 統合
  ├── テツヤ: 図鑑画面、タイマー、保護者ダッシュボード
  ├── ハルカ: ユーザビリティ検証、UI調整
  ├── リカ: テストユーザー募集、NPS調査設計
  └── ケンジ: QAテスト、エッジケース検証

Week 4: ユーザーテスト
  ├── ユウキ: 5組の親子テスト実施・観察
  ├── ハルカ: UXフィードバック分析
  ├── マナブ: 教育効果の初期評価
  └── 全員: 振り返り・改善点洗い出し

Week 5-6: 改善 + ローンチ準備
  ├── テスト結果に基づく修正
  ├── 法務レビュー完了
  ├── 3-4歳版の追加（必要に応じて）
  └── Phase 0 ローンチ
```

## 9. チーム議論の振り返り

### 生産的摩擦が生んだ成果

| 摩擦 | 結果 |
|------|------|
| ユウキ(楽観) vs テツヤ(懐疑) | 図鑑アニメ → 静止画に現実化。実現可能なMVPに |
| ハルカ(感情) vs テツヤ(ひらがな) | 対立から複合モジュールが誕生 |
| マナブ(教育) vs リカ(市場) | SELの教育価値を保ちつつ、市場ニーズ(ひらがな)と統合 |
| ケンジ(批判) vs 全員 | 法務・タイマー・効果測定が最初から組み込まれた |
| リカ(英語) vs マナブ(母語優先) | 英語はPhase 2に段階化、発達段階を尊重 |
