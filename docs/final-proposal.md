# 子供向け学習Webアプリ — 最終企画書（個人プロジェクト版）

> **プロジェクト名**: そだてる ずかん（仮称）
> **ターゲット**: うちの子（3〜6歳）
> **用途**: 家庭内利用のみ
> **策定日**: 2026-02-28
> **策定方法**: 6名AIチームによる3ラウンド議論 → 個人プロジェクトへ方針転換

---

## 1. コンセプト

### 「育てる図鑑 × きもちの冒険」

学ぶたびに図鑑のページが増え、キャラクターに「きもちのことば」を教えてあげる体験。
コレクションの喜びと、感情教育（SEL）を自然に融合させた、うちの子専用学習アプリ。

**コアバリュー**:
- 子供が「教える側」になることで学びが定着する（プロテジェ効果）
- 正解を押し付けず、共感と発見で動機づける
- 親子の関わりを技術で補助する

**個人プロジェクトならではの強み**:
- うちの子の名前・好きなもの・興味をハードコードでき、完全カスタマイズ可能
- リビングがテスト会場。反応を見て即改善できる最高のフィードバックループ
- 市場に合わせる必要なし。純粋に教育的に正しいことだけを追求できる
- 「完成」を目指さず、子供の成長に合わせてアプリも育てていく

## 3. MVP仕様

### 3.1 学習モジュール（1本）

**ひらがな × きもちことば 複合モジュール**

```
フロー例:
1. キャラ（くまの「モコ」）が画面に登場
2. モコが「今日は"あ"のつくことばを探そう！」と音声で話す
3. 選択肢が3つ表示（絵＋ひらがな）→ タップで回答
4. 正解 → 図鑑ページが1枚開く！（トランジション演出）
5. 開いた図鑑のページでモコが気持ちを表現
   「あめが降って、モコはちょっとかなしいな...」
6. 子供に問いかけ:「モコにどんなきもちを教えてあげる？」
7. 感情アイコンをタップ → どれを選んでもモコが反応
   「そうなんだね、教えてくれてありがとう！」
8. 1セッション = 5問（約5分）→ やさしく終了
```

### 3.2 年齢別設計

| 要素 | 3-4歳 | 5-6歳 |
|------|-------|-------|
| ひらがな | 5文字（あいうえお） | 10文字〜 |
| 感情選択肢 | 2択（うれしい/かなしい） | 4択（+こわい/おこった） |
| キャラの問いかけ | 「えらんでね」 | 「どうしてそう思う？」 |
| 図鑑ページ | 5ページ | 10ページ |
| UIサイズ | 特大ボタン | 大ボタン |

> 個人プロジェクトなので、子供の実際の反応を見ながら柔軟に調整可能。
> 年齢設定は目安であり、子供の発達に合わせて自由に切り替えられる。

### 3.3 パーソナライズ要素（個人プロジェクトならでは）

- 子供の名前で呼びかけ:「○○ちゃん、今日もモコと遊ぼう！」
- 子供が好きな動物・食べ物・色を図鑑テーマに反映
- 家族の写真やイラストを素材として使える
- 季節や行事に合わせたコンテンツ追加が自由

### 3.4 保護者機能（シンプル版）

- 進捗確認（localStorageのデータをコンソールで確認 or シンプルな親画面）
- スクリーンタイム制限（1日の上限設定）
- 学習履歴のJSONエクスポート（データ消失対策）

### 3.5 実装しないもの（MVP除外）

- ❌ 音声認識
- ❌ AI個別最適化（親が直接調整すればいい）
- ❌ ユーザー認証・アカウント管理
- ❌ バックエンド / データベース
- ❌ リッチアニメーション（初期）
- ❌ ソーシャル機能

## 4. 技術スタック（個人プロジェクト最小構成）

| レイヤー | 技術 | 選定理由 |
|---------|------|---------|
| フロントエンド | **React (Vite)** | 軽量・高速。Next.jsほどの機能は不要 |
| アニメーション | **Framer Motion** | タップ操作・トランジションに十分 |
| スタイリング | **Tailwind CSS** | 迅速なUI構築 |
| データ保存 | **localStorage** | サーバー不要。家庭内利用なら十分 |
| 音声再生 | **Web Audio API** | ブラウザ標準、追加依存なし |
| ホスティング | **GitHub Pages** or **ローカル** | 無料。`npm run dev` でも十分 |
| 素材 | **プロジェクト内static** | 画像・音声はリポジトリに直接含める |

### テツヤの技術評価（方針転換後）

> **「認証もDBもサーバーも不要。React + localStorage、これだけ。」**
>
> - 1人の週末作業で **1〜2週間** でMVP完成可能
> - 子供の名前や好みをハードコードして演出に使える
> - 実験的なUI、効果音の遊びなど「業務では後回しにする楽しいこと」を自由に試せる
> - ただし以下は守るべき:
>   - **タブレット対応**（子供はタブレットで使う可能性大）
>   - **データ消失対策**（localStorage は消えうる → JSONエクスポート機能は必須）
>   - **コード整理**（半年後の自分は他人だ）

## 5. 教育的裏付け（教育専門家 マナブ承認済み）

### 教育的妥当性: **9.0 / 10**（個人版は+0.5。子供に完全最適化できるため）

| 理論 | 適用箇所 |
|------|---------|
| **ピアジェ 前操作期** | シンボルと言語の獲得段階に合わせたひらがな＋感情語彙 |
| **ヴィゴツキー ZPD** | 子供の反応を直接見て「最近接発達領域」を動的に把握・調整 |
| **プロテジェ効果** | キャラに「教えてあげる」ことで学びが定着 |
| **SEL（社会情動的学習）** | 感情語彙の習得が認知発達の土台になる |
| **内発的動機づけ** | 感情パートは正解なし、共感と受容がベース |
| **個別最適化学習** | 親が開発者＝子供の発達に完全に合わせられる（商業製品では不可能） |

### 感情パートの設計原則

> **「否定せず、視野を広げる」方式**
> - 子供がどの感情を選んでも「そうなんだね、教えてくれてありがとう」と受容
> - その上でキャラが「ぼくはこう感じるよ」と別の視点を提示
> - テストではなく対話

### 効果測定（個人版）

- 子供がアプリで出た感情の言葉を日常で使い始めたかを親が直接観察
- 図鑑コンプリートへの子供の反応を直接確認
- ひらがなの定着度は日常の中で自然にわかる

## 6. 気をつけること（QA ケンジの助言・個人版）

### 法的対応
> 個人利用のため法務レビューは不要。ただし将来公開する場合は再検討。

### それでも守るべきこと

| # | 項目 | 理由 |
|---|------|------|
| 1 | **スクリーンタイム制限** | WHOガイドライン: 2-4歳は1日1時間。5分タイマーは有効 |
| 2 | **ブルーライト配慮** | 寝る前の使用は避ける。ダークモードやナイトシフト対応 |
| 3 | **音量制限** | 子供の耳を守る。音声再生は適切な音量に固定 |
| 4 | **データバックアップ** | localStorageは消えうる。定期的なJSONエクスポート |
| 5 | **中毒性の回避** | 無限にプレイさせない設計。「また明日ね」の終了画面 |

## 7. 開発スケジュール（個人プロジェクト版）

```
Weekend 1: 基盤づくり
  ├── Vite + React プロジェクト初期化
  ├── 基本レイアウト（タブレット対応）
  ├── キャラ「モコ」の表示（プレースホルダ画像）
  └── タップ操作の基本実装

Weekend 2: コア機能
  ├── ひらがなクイズ（5文字分）
  ├── 感情選択UI
  ├── 図鑑ページ（静止画＋トランジション）
  └── localStorage で進捗保存

Weekend 3: 仕上げ + 子供に触らせる
  ├── 音声再生（読み上げ・効果音）
  ├── 5分タイマー・終了画面
  ├── 子供の名前パーソナライズ
  └── 🎉 うちの子に触らせてみる！

その後: 子供の反応を見ながら育てる
  ├── 好きなテーマの図鑑ページ追加
  ├── ひらがな追加（か行、さ行...）
  ├── 感情語彙の拡張
  ├── 気に入ったら少しずつアニメーション追加
  └── 子供の成長に合わせて難易度調整
```
